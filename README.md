# SourceLens

**Chat with your own books and documents.** SourceLens is a free desktop app for Windows and Linux
that lets you build a personal library of PDF, EPUB and text files and ask questions about them in
plain language — typed or by voice. Every answer is backed by quotes from your documents, so you can
always see exactly where the information came from.

Made for students, researchers and anyone who works with a lot of reading material: textbooks,
papers, manuals, notes.

## What it can do

- **Your personal library** — add books and articles (`.pdf`, `.epub`, `.txt`, `.md`) right from the
  app and it indexes them for search. No size limits, no uploads to any server: the index lives on
  your computer.
- **Questions in plain language** — type your question or just speak it; voice is transcribed
  locally on your machine (Whisper).
- **Answers with receipts** — along with the answer you get the exact passages it was based on,
  with relevance scores, one-click summaries and full-text view.
- **A real conversation** — ask follow-up questions; the app remembers the context. The whole
  history is saved and browsable, even after a restart.
- **Choice of AI** — answers are generated by an AI assistant you already have: **Claude** or
  **Codex**. Switch the engine and model anytime in Settings.

## Getting started (no programming required)

### 1. Download

Grab the latest build from the [Releases page](../../releases):

- **Windows:** `SourceLens-…-win-x64.zip` — unpack anywhere and run `SourceLens.exe`.
- **Linux:** `SourceLens-…-linux-x64.tar.gz` — unpack and run `./SourceLens`.

Nothing else to install for the app itself — the builds are self-contained.

### 2. One thing you do need: an AI assistant

SourceLens doesn't include its own AI — it asks one you already use. Install and sign in to at
least one of these (in a terminal, once):

- **Claude Code** (`claude`) — [installation guide](https://docs.anthropic.com/en/docs/claude-code/setup)
- **OpenAI Codex CLI** (`codex`) — [installation guide](https://github.com/openai/codex)

The app simply runs the assistant on your behalf using your existing sign-in. It never asks for or
stores API keys.

Optional, Linux only: install **ffmpeg** (`sudo apt install ffmpeg` or similar) if you want voice
input. On Windows voice recording works out of the box.

### 3. First launch

On the first run the app downloads two helper models into its folder (about 600 MB total, internet
required once): a local search model for your library and a local speech-recognition model. After
that it works offline except for the AI answers themselves.

## How to use it

1. Click **Sources** → **Add sources** and pick your files. You'll see indexing progress right
   there; the same window lets you remove documents later. (Files are copied into the app's
   library, so you can move or delete the originals afterwards.)
2. Type a question in the box at the bottom (**Ctrl+Enter** to send) — or press the microphone
   button and ask out loud.
3. Read the answer and check the cited passages under it: each shows where it came from and how
   relevant it is, with **Summarize** / **Show full** buttons.
4. Keep the conversation going — follow-ups understand the context. Past sessions are listed in
   the left panel; you can reopen any of them or re-ask an old question.
5. In **Settings**, choose which assistant answers (Claude or Codex), pick a model (the list is
   detected from the assistant itself) and, if needed, point the app to where the assistant is
   installed. Your choices are remembered.

**Privacy:** your documents are indexed and searched entirely on your computer, and voice is
transcribed locally too. Only your question and the few passages selected as relevant are sent to
the AI engine you chose.

## For developers

Everything below is only needed if you want to build SourceLens from source or hack on it.

### Requirements

- **.NET SDK** 9/10 (projects target `net9.0`).
- Claude CLI / Codex CLI and ffmpeg as above for actually running the app.

### Build & test

```bash
cd SourceLens
dotnet build src/SourceLens.sln
dotnet run --project src/SourceLens/SourceLens.csproj

dotnet test src/SourceLens.Tests/SourceLens.Tests.csproj            # unit + headless UI tests
# integration tests (download models / probe real CLIs), run explicitly:
dotnet test src/SourceLens.Tests/SourceLens.Tests.csproj --filter "Category=Integration"
# end-to-end RAG pipeline (real SQLite + real ONNX embedder):
dotnet test src/SourceLens.Tests/SourceLens.Tests.csproj --filter "FullyQualifiedName~EndToEndTests"
```

Releases are produced automatically by GitHub Actions on every push to `main`
(`.github/workflows/dotnet-desktop.yml`): tests, then self-contained `win-x64` / `linux-x64`
publishes attached to a GitHub Release.

### How it works

Documents are chunked and embedded locally with the `multilingual-e5-small` ONNX model, vectors are
stored in SQLite (`sourcelens.db` next to the binary); at question time the top-K passages by
cosine similarity are injected into the prompt of a CLI agent (Claude CLI / Codex CLI), which
produces the answer. Voice input is ffmpeg/NAudio recording + whisper.cpp (Whisper.net)
transcription. Dialog history with source snapshots is persisted in the same database.

Files created at runtime next to the binary: `sourcelens.db` (index + dialog history + settings),
`models/`, `books/` (the library; files added via the UI are copied here, deduplicated by SHA-256;
the folder is also scanned recursively on startup), `logs/`.

### Configuration (`appsettings.json`)

`appsettings.template.json` is copied to `appsettings.json` next to the binary on first start
(the working copy is gitignored).

| Section | Keys | Notes |
|---|---|---|
| `AiModel` | `Provider` (`Claude`/`Codex`/`Disabled`), per-engine `BinaryPath`, `ExtraArgs`, `DefaultModel`, `TimeoutSeconds` | `Provider`, `DefaultModel` and `BinaryPath` are only the initial defaults; the in-app Settings choices (stored in the database) take priority. Available models are discovered from the CLI at runtime, not configured. |
| `Rag` | `Enabled`, `BooksFolder`, `TopK`, `MinQueryLength`, `ChunkerVersion`, `ChunkSize`, `ChunkOverlap`, `HistoryDepth`, `MaxHistoryChars`, `EmbeddingProvider`, `LocalOnnx{ModelIdLabel,Dimensions,MaxSequenceLength}` | `HistoryDepth=0` disables dialog context (history is still saved). Changing `ChunkerVersion`/embedder settings triggers reindexing. |
| `Transcription` | `Model` (`Tiny`/`Base`/`Small`/`Medium`/`Large`), `Language` (`auto`), `UseGpu`, `Threads`, `PoolSize` | CPU by default. |
| `Audio` | `SourceName`, `Rate`, `Channels`, `BitsPerSample` | `SourceName` is a PulseAudio source on Linux (`pactl list short sources`; `default` works). |

Invalid configuration is reported in a startup error window and in `logs/Error.log`.
